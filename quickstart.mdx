## Get started in minutes

This guide will walk you through creating a new Hive project, running your first API endpoint, and understanding the core concepts.

<Steps>
  <Step title="Install Hive CLI">
    Install the Hive CLI globally using npm:

    ```bash
    npm install -g @paralect/hive
    ```

    <Note>
      Make sure you have Node.js 16+ and MongoDB running locally or accessible via connection string.
    </Note>
  </Step>

  <Step title="Create a new project">
    Initialize a new Hive project:

    ```bash
    hive init my-app
    cd my-app
    ```

    This creates a minimal project structure:

    ```
    my-app/
    ├── src/
    │   ├── resources/      # Your features
    │   ├── middlewares/    # Custom middlewares
    │   ├── services/       # External APIs
    │   └── scheduler/handlers/
    ├── .env                # Environment variables
    ├── .gitignore
    └── package.json
    ```

    <Tip>
      The `.hive/` folder is auto-generated when you run the server. Never edit it directly.
    </Tip>
  </Step>

  <Step title="Configure environment">
    Open `.env` and verify your MongoDB connection:

    ```bash .env
    MONGO_URI=mongodb://localhost:27017/my-app
    PORT=3001
    NODE_ENV=development
    ```

    <Warning>
      Make sure MongoDB is running before starting the server. You can start it with `mongod` or use a cloud provider like MongoDB Atlas.
    </Warning>
  </Step>

  <Step title="Start the development server">
    Run your Hive application:

    ```bash
    hive run ./src
    ```

    You should see:

    ```
    🐝 Hive server starting...
    Api server listening on 3001, in development mode
    ```

    Your API is now live at `http://localhost:3001`
  </Step>

  <Step title="Create your first resource">
    Let's create a `tasks` resource. Create a schema file:

    ```bash
    mkdir -p src/resources/tasks
    ```

    Create `src/resources/tasks/tasks.schema.js`:

    ```javascript src/resources/tasks/tasks.schema.js
    import { z } from 'zod';
    import dbSchema from 'helpers/schema/db.schema.js';

    const schema = dbSchema.extend({
      title: z.coerce.string().nullable().optional(),
      status: z.coerce.string().default('pending'),
      assignee: z.object({
        _id: z.string(),
        fullName: z.coerce.string().nullable().optional(),
      }).nullable().optional(),
    });

    export default schema;
    export const secureFields = [];
    ```

    <Note>
      Every schema automatically gets `_id`, `createdOn`, and `updatedOn` fields from `dbSchema`.
    </Note>
  </Step>

  <Step title="Create a list endpoint">
    Create the endpoints directory:

    ```bash
    mkdir -p src/resources/tasks/endpoints
    ```

    Create `src/resources/tasks/endpoints/list.js`:

    ```javascript src/resources/tasks/endpoints/list.js
    import { z } from 'zod';
    import db from 'db';

    const tasksService = db.services.tasks;

    export const handler = async (ctx) => {
      const { page, perPage } = ctx.validatedData;
      return tasksService.find({}, { page, perPage, sort: '-createdOn' });
    };

    export const middlewares = [];

    export const requestSchema = z.object({
      page: z.coerce.number().default(1),
      perPage: z.coerce.number().default(20),
    });

    export const endpoint = {
      url: '/',
      method: 'get',
    };
    ```

    <Tip>
      The route is auto-generated as `GET /tasks` based on the resource name and endpoint config.
    </Tip>
  </Step>

  <Step title="Create a task endpoint">
    Create `src/resources/tasks/endpoints/create.js`:

    ```javascript src/resources/tasks/endpoints/create.js
    import { z } from 'zod';
    import db from 'db';
    import isAuthorized from 'middlewares/isAuthorized';

    const tasksService = db.services.tasks;

    export const handler = async (ctx) => {
      const task = await tasksService.create({
        ...ctx.validatedData,
        assignee: ctx.state.user,
      });
      return task;
    };

    export const middlewares = [isAuthorized];

    export const requestSchema = z.object({
      title: z.string(),
      status: z.string().default('pending'),
    });

    export const endpoint = {
      url: '/',
      method: 'post',
    };
    ```

    This endpoint requires authentication using the built-in `isAuthorized` middleware.
  </Step>

  <Step title="Test your API">
    Restart the server (Ctrl+C and `hive run ./src`). Test the list endpoint:

    <CodeGroup>
      ```bash cURL
      curl http://localhost:3001/tasks
      ```

      ```javascript JavaScript
      const response = await fetch('http://localhost:3001/tasks');
      const data = await response.json();
      console.log(data);
      ```

      ```python Python
      import requests

      response = requests.get('http://localhost:3001/tasks')
      print(response.json())
      ```
    </CodeGroup>

    You should receive:

    ```json
    {
      "results": [],
      "count": 0
    }
    ```
  </Step>
</Steps>

## AI-native development

If you're using Cursor or another AI editor, you can build even faster with natural language commands:

```
add-resource projects
add-endpoint projects post name, description
add-endpoint projects get page, perPage
add-handler projects
add-middleware isProjectOwner
```

The AI understands Hive conventions and generates production-ready code.
