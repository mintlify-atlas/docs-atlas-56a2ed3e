## What's preconfigured

| Concern | Solution |
|---------|----------|
| Database | MongoDB with auto-generated services per schema |
| Validation | Zod schemas, auto-applied to all endpoints |
| Auth | Token-based auth infrastructure, ready to extend |
| Events | Persistent handlers that react to DB changes |
| Real-time | Socket.io with Redis adapter for scaling |
| Queues | BullMQ for background jobs |
| Scheduler | Cron-based jobs, no external service needed |
| Auto-sync | Embedded documents stay fresh automatically |

  Hive uses proven, boring technology: Koa, MongoDB, Zod, Socket.io, BullMQ, and Redis.

## Architecture overview

Every feature in Hive is a **resource** — a folder with a predictable structure:

```
src/resources/{name}/
├── {name}.schema.js      # Data shape (required)
├── endpoints/            # API routes
├── handlers/             # React to DB events
└── methods/              # Shared logic
```

This architecture scales. New developers understand it in minutes. Your codebase stays clean at 10 endpoints or 1000.

## Philosophy

> Technology should solve people's issues, not create new weird ones.

Hive is built for engineers who prioritize:
- **Product outcomes** over technical purity
- **Minimal boilerplate** and cognitive overhead
- **Challenging complexity** — if it doesn't serve the user, remove it
- **Less code** = better outcome

## Tech stack

Hive is built on proven, production-ready technologies:

- **Koa** — Lightweight Node.js framework
- **MongoDB** — Document database with flexible schemas
- **Zod** — TypeScript-first schema validation
- **Socket.io** — Real-time bidirectional communication
- **BullMQ** — Redis-based job queues
- **Redis** — Caching and pub/sub
