Hive uses [Zod](https://zod.dev) for runtime schema validation and type safety. Every schema is validated before database operations.

## How validation works

Validation happens automatically at two levels:

1. **Request validation** - Validates incoming request data using the `validate` middleware
2. **Database validation** - Validates data before create/update operations

## Request validation

Use the `validate` middleware to validate incoming requests against a Zod schema:

```javascript
import validate from 'middlewares/validate';
import { z } from 'zod';

const createUserSchema = z.object({
  email: z.string().email(),
  fullName: z.string(),
  password: z.string().min(8),
});

export default {
  method: 'post',
  path: '/users',
  middlewares: [validate(createUserSchema)],
  handler: async (ctx) => {
    // Access validated data
    const { email, fullName, password } = ctx.validatedData;
    
    // Create user...
  },
};
```

### Validation sources

The `validate` middleware automatically merges and validates data from:

- `ctx.request.body` - Request body
- `ctx.query` - Query parameters  
- `ctx.params` - URL parameters

```javascript middlewares/validate.js
const validate = (schema) => async (ctx, next) => {
  const result = await schema.passthrough().safeParseAsync({
    ...(ctx.request.body),
    ...ctx.query,
    ...ctx.params,
  });

  if (!result.success) ctx.throw(400, { clientErrors: formatError(result.error) });

  ctx.validatedData = result.data;

  await next();
};
```

<Note>
The middleware uses `.passthrough()` to allow additional fields not defined in the schema. This is useful for extending schemas in user code.
</Note>

## Error formatting

Validation errors are automatically formatted into a user-friendly structure:

```javascript
const formatError = (zodError) => {
  const errors = {};

  zodError.issues.forEach((error) => {
    const key = error.path.join('.');

    if (!errors[key]) {
      errors[key] = [];
    }

    (errors[key]).push(error.message);
  });

  return errors;
};
```

### Error response format

When validation fails, the API returns a 400 error with structured error messages:

```json
{
  "clientErrors": {
    "email": ["Invalid email"],
    "password": ["String must contain at least 8 character(s)"]
  }
}
```

<Warning>
Validation errors automatically return a 400 status code. The error object is nested under `clientErrors` to distinguish from server errors.
</Warning>

## Database validation

Every database service includes automatic schema validation before writes:

```javascript db.js
db.services[schemaName] = db.createService(`${resourceName}`, {
  validate: (obj) => {
    return { value: schema.parse(obj) };
  },
  secureFields: secureFields,
});
```

### Validation on create

```javascript
async create(objs, options = {}) {
  const entities = _.isArray(objs) ? objs : [objs];

  let created = await Promise.all(
    entities.map(async (doc) => {
      const entity = _.cloneDeep(doc);

      if (this._options.useStringId && !entity._id)
        entity._id = this.generateId();
      if (this._options.addCreatedOnField && !entity.createdOn) {
        entity.createdOn = new Date().toISOString();
      }
      const validated = await this._validate(entity);

      return validated;
    })
  );

  await this._collection.insert(created, options);
  // ...
}
```

### Validation on update

```javascript
async updateOne(query, updateFn, options = {}) {
  const doc = await this.findOne(query, { isIncludeSecureFields: true });
  
  let entity = _.cloneDeep(doc);
  
  if (this._options.addUpdatedOnField)
    entity.updatedOn = new Date().toISOString();
  entity = await updateFn(entity);
  let updated = await this._validate(entity);

  await this._collection.update(
    { ...query, _id: doc._id },
    { $set: updated },
    options
  );
  // ...
}
```

## Schema validation errors

When database validation fails, a `MongoServiceError` is thrown:

```javascript
async _validate(entity) {
  if (this._options.validate) {
    const { value, error } = await this._options.validate(entity);

    if (error) {
      console.log("Schema Error", error);

      throw new MongoServiceError(
        MongoServiceError.INVALID_SCHEMA,
        `Document schema is invalid: ${JSON.stringify(error, null, 2)}`,
        error
      );
    }

    return value;
  }

  return entity;
}
```

<Tip>
Database validation uses `.parse()` which throws on error, while request validation uses `.safeParseAsync()` for error handling.
</Tip>

## Common validation patterns

### Email validation

```javascript
z.string().email()
```

### URL validation

```javascript
z.string().url()
```

### Optional fields

```javascript
z.string().optional()
z.string().nullable().optional()
```

### Nested objects

```javascript
z.object({
  user: z.object({
    _id: z.string(),
  }),
})
```

### Enums

```javascript
z.enum(['asc', 'desc'])
```

### Coercion

```javascript
z.coerce.number()  // Convert string to number
z.coerce.date()    // Convert string to Date
```

### Defaults

```javascript
z.string().default('default value')
z.coerce.date().default(() => new Date())
```

## Pagination schema example

Here's a complete example from the framework:

```javascript helpers/schema/pagination.schema.js
import { z } from 'zod';

export const paginationSchema = z.object({
  page: z.coerce.number().default(1),
  perPage: z.coerce.number().default(10),

  searchValue: z.string().optional(),

  sort: z
    .object({
      createdOn: z.enum(['asc', 'desc']).default('asc'),
    })
    .default({}),
});
```

This schema:
- Coerces page numbers from strings to numbers
- Provides sensible defaults (page 1, 10 items per page)
- Makes search optional
- Validates sort direction with an enum

## Extending schemas

You can extend framework schemas in your user code:

```javascript src/resources/users/users.extends.schema.js
import { z } from 'zod';

export default z.object({
  phoneNumber: z.string().optional(),
  role: z.enum(['admin', 'user']).default('user'),
});
```

The framework automatically merges extended schemas:

```javascript db.js
if (process.env.HIVE_SRC) {
  let extendSchemaPath = `${process.env.HIVE_SRC}/resources/${resourceName}/${schemaName}.extends.schema.js`;
  if (fs.existsSync(extendSchemaPath)) {
    let extendsSchema = (await import(extendSchemaPath)).default;
    schema = schema.append(extendsSchema);
  }
}
```
