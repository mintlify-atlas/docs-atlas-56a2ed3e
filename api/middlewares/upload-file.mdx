## Overview

The `uploadFile` middleware enables file uploads in your endpoints using `@koa/multer`. Files are stored in memory as buffers, ready for processing or upload to cloud storage.

## Usage

```javascript
import uploadFile from 'middlewares/uploadFile';

export const middlewares = [uploadFile.single('file')];

export const handler = async (ctx) => {
  const file = ctx.file; // Access uploaded file
  
  // file.buffer contains the file data
  // file.originalname contains the filename
  // file.mimetype contains the content type
  
  return { uploaded: true, filename: file.originalname };
};
```

## Multer methods

The middleware is a configured multer instance with memory storage. Use these methods:

<ParamField path=".single(fieldName)" type="function">
  Accept a single file with the given field name. The file will be available at `ctx.file`.
  
  ```javascript
  uploadFile.single('avatar')
  ```
</ParamField>

<ParamField path=".array(fieldName, maxCount)" type="function">
  Accept an array of files with the given field name. Files will be available at `ctx.files`.
  
  ```javascript
  uploadFile.array('photos', 10) // Max 10 files
  ```
</ParamField>

<ParamField path=".fields(fields)" type="function">
  Accept multiple fields with different names. Files organized by field name in `ctx.files`.
  
  ```javascript
  uploadFile.fields([
    { name: 'avatar', maxCount: 1 },
    { name: 'gallery', maxCount: 8 }
  ])
  ```
</ParamField>

<ParamField path=".none()" type="function">
  Accept only text fields, no files. Useful for multipart/form-data without files.
  
  ```javascript
  uploadFile.none()
  ```
</ParamField>

## File object structure

When a file is uploaded, it's available on `ctx.file` or `ctx.files` with these properties:

<ResponseField name="fieldname" type="string">
  Field name specified in the form
</ResponseField>

<ResponseField name="originalname" type="string">
  Name of the file on the user's computer
</ResponseField>

<ResponseField name="encoding" type="string">
  Encoding type of the file (e.g., "7bit")
</ResponseField>

<ResponseField name="mimetype" type="string">
  MIME type of the file (e.g., "image/jpeg", "application/pdf")
</ResponseField>

<ResponseField name="buffer" type="Buffer">
  The file data as a Buffer. This is the actual file content in memory.
</ResponseField>

<ResponseField name="size" type="number">
  Size of the file in bytes
</ResponseField>

## Examples

### Single file upload

Upload a user avatar:

```javascript
import { z } from 'zod';
import uploadFile from 'middlewares/uploadFile';
import isAuthorized from 'middlewares/isAuthorized';
import db from 'db';

export const middlewares = [isAuthorized, uploadFile.single('avatar')];

export const requestSchema = z.object({});

export const handler = async (ctx) => {
  const file = ctx.file;
  
  if (!file) {
    ctx.throw(400, 'No file uploaded');
  }

  // Validate file type
  const allowedTypes = ['image/jpeg', 'image/png', 'image/gif'];
  if (!allowedTypes.includes(file.mimetype)) {
    ctx.throw(400, 'Invalid file type. Only JPEG, PNG, and GIF allowed.');
  }

  // Validate file size (max 5MB)
  const maxSize = 5 * 1024 * 1024;
  if (file.size > maxSize) {
    ctx.throw(400, 'File too large. Maximum size is 5MB.');
  }

  // Upload to cloud storage (example with AWS S3)
  const s3Key = `avatars/${ctx.state.user._id}-${Date.now()}.${file.originalname.split('.').pop()}`;
  // await uploadToS3(file.buffer, s3Key, file.mimetype);

  // Update user with avatar URL
  await db.services.users.updateOne(
    { _id: ctx.state.user._id },
    (doc) => ({ ...doc, avatarUrl: `https://cdn.example.com/${s3Key}` })
  );

  return { 
    uploaded: true, 
    filename: file.originalname,
    size: file.size 
  };
};

export const endpoint = {
  url: '/avatar',
  method: 'post',
};
```

### Multiple files

Upload multiple images:

```javascript
import uploadFile from 'middlewares/uploadFile';

export const middlewares = [uploadFile.array('images', 5)];

export const handler = async (ctx) => {
  const files = ctx.files;
  
  if (!files || files.length === 0) {
    ctx.throw(400, 'No files uploaded');
  }

  const uploadedFiles = files.map(file => ({
    name: file.originalname,
    size: file.size,
    type: file.mimetype
  }));

  // Process each file
  for (const file of files) {
    // Upload to storage, create thumbnails, etc.
  }

  return { 
    uploaded: files.length,
    files: uploadedFiles 
  };
};

export const endpoint = {
  url: '/gallery',
  method: 'post',
};
```

### CSV import

Parse uploaded CSV:

```javascript
import uploadFile from 'middlewares/uploadFile';
import csv from 'csv-parse/sync';

export const middlewares = [uploadFile.single('csv')];

export const handler = async (ctx) => {
  const file = ctx.file;
  
  if (!file) {
    ctx.throw(400, 'No CSV file uploaded');
  }

  if (file.mimetype !== 'text/csv') {
    ctx.throw(400, 'File must be CSV');
  }

  // Parse CSV
  const records = csv.parse(file.buffer.toString(), {
    columns: true,
    skip_empty_lines: true
  });

  // Import records
  const imported = [];
  for (const record of records) {
    const created = await db.services.contacts.create({
      name: record.name,
      email: record.email,
      phone: record.phone
    });
    imported.push(created);
  }

  return { 
    imported: imported.length,
    total: records.length 
  };
};
```

## Implementation

From `src/middlewares/uploadFile.js`:

```javascript
import multer from '@koa/multer';

const storage = multer.memoryStorage();

export default multer({ storage });
```

The middleware uses memory storage, so files are stored as Buffers in RAM. This is ideal for:
- Small to medium files
- Files that will be immediately processed or uploaded elsewhere
- Serverless environments

## Memory considerations

<Warning>
  Files are stored in memory. For large files or high traffic, consider:
  - Limiting file sizes with multer options
  - Using disk storage for large files
  - Processing files in background jobs
  - Streaming directly to cloud storage
</Warning>

## Configuration options

You can create a custom configured multer instance:

```javascript
import multer from '@koa/multer';

const upload = multer({
  storage: multer.memoryStorage(),
  limits: {
    fileSize: 10 * 1024 * 1024, // 10MB
    files: 5 // Max 5 files
  },
  fileFilter: (req, file, cb) => {
    // Only accept images
    if (file.mimetype.startsWith('image/')) {
      cb(null, true);
    } else {
      cb(new Error('Only images allowed'));
    }
  }
});

export const middlewares = [upload.single('photo')];
```

## Error handling

```javascript
export const handler = async (ctx) => {
  try {
    const file = ctx.file;
    
    if (!file) {
      ctx.throw(400, 'No file uploaded');
    }

    // Process file
    
  } catch (error) {
    if (error.code === 'LIMIT_FILE_SIZE') {
      ctx.throw(400, 'File too large');
    }
    if (error.code === 'LIMIT_FILE_COUNT') {
      ctx.throw(400, 'Too many files');
    }
    throw error;
  }
};
```
